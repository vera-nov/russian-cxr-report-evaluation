# Оценка качества русскоязычных радиологических отчётов

Воспроизводимый пайплайн для построения и применения метода оценки качества русскоязычных радиологических отчётов.

Метод состоит из следующих компонент:
1. собственная реализация метода GREEN на русском языке;
2. англоязычная текстовая компонента, вычисляемая после перевода русскоязычных отчётов на английский язык;
3. визуально-текстовая компонента, оценивающую согласованность отчёта-кандидата с рентгеновским снимком.

Итоговая оценка качества вычисляется как линейная комбинация нормированных компонент:

$$
S = w_{ru} z_{ru} + w_{text} z_{text} + w_{img} z_{img},
$$

где $z_{ru}$, $z_{text}$ и $z_{img}$ — стандартизованные значения базовых компонент, а $w_{ru}$, $w_{text}$, $w_{img}$ — веса смешивания. Веса и конкретные базовые методы выбираются автоматически по обучающей выборке с помощью вложенной кросс-валидации и оптимизации корреляции Кендалла с экспертной оценкой.

## Состав репозитория

| Файл | Назначение |
|---|---|
| `build_method.py` | Полный этап построения метода: подготовка данных, перевод отчётов, расчёт базовых компонент, вложенная кросс-валидация, выбор лучшей конфигурации и сохранение JSON-конфига. |
| `evaluate.py` | Применение ранее сохранённого JSON-конфига к новым данным и сохранение итоговых оценок качества. |
| `translate_reports.py` | Перевод русскоязычных отчётов на английский язык. |
| `calculate_russian_text.py` | Расчёт русскоязычной GREEN-like компоненты. |
| `calculate_english_text.py` | Расчёт текстовых англоязычных компонент. |
| `calculate_image_text.py` | Расчёт визуально-текстовой компоненты. |
| `utils.py` | Общие функции: нормализация входных данных, описание базовых методов, построение конфигураций, подбор весов, кросс-валидация и применение сохранённого конфига. |

## Входные данные

На вход подаётся CSV-файл. Минимально необходимы следующие столбцы:

| Столбец | Описание |
|---|---|
| кандидатный отчёт | Сгенерированный или оцениваемый радиологический отчёт. |
| эталонный отчёт | Референсный отчёт, с которым сравнивается кандидат. |
| путь к изображению | Путь к рентгеновскому снимку. |
| целевая оценка | Числовая экспертная оценка качества, например число клинически значимых ошибок. |

Если пути к изображениям в CSV указаны относительно некоторой директории, её можно передать через `--image-root`.

## Построение метода

Скрипт `build_method.py` вычисляет базовые компоненты, перебирает допустимые конфигурации и подбирает веса их линейной комбинации. Результат сохраняется в JSON-файл, который затем используется на этапе оценки.

Пример запуска:

```bash
uv run python build_method.py \
  --candidate-col M2Tr-Generation_ru \
  --reference-col ground_truth_ru \
  --image-col path_to_image \
  --target-col SIG_error_count \
  --gpu 3 \
  --device cuda \
  --verbose \
  --target-direction lower \
  --path-to-data-description test.csv
```

По умолчанию конфигурация сохраняется в файл:

```text
quality_method_config.json
```

### Основные параметры `build_method.py`

| Параметр | Описание |
|---|---|
| `--path-to-data-description` | Путь к CSV-файлу с обучающими данными. |
| `--candidate-col` | Название столбца с отчётом-кандидатом. |
| `--reference-col` | Название столбца с эталонным отчётом. |
| `--image-col` | Название столбца с путём к изображению. |
| `--target-col` | Название столбца с экспертной числовой оценкой. |
| `--output-json` | Путь для сохранения итогового JSON-конфига. По умолчанию `quality_method_config.json`. |
| `--target-direction` | Направление целевой переменной: `higher`, если больше — лучше; `lower`, если меньше — лучше. Для числа ошибок используйте `lower`. |
| `--translators` | Список переводчиков через запятую. По умолчанию `qwen,hy_mt,translategemma`. |
| `--translation-kinds` | Режимы перевода через запятую. По умолчанию `terms,noterms`. |
| `--k-outer` | Число внешних фолдов вложенной кросс-валидации. По умолчанию `5`. |
| `--k-inner` | Число внутренних фолдов. По умолчанию `4`. |
| `--weight-step` | Шаг сетки для перебора весов. По умолчанию `0.25`. Значение должно делить 1.0 без остатка, например `0.25`, `0.1`, `0.05`. |
| `--random-state` | Seed для воспроизводимого разбиения на фолды. |
| `--gpu` | Номер GPU. |
| `--device` | Устройство вычислений, например `cuda` или `cpu`. |
| `--image-root` | Корневая директория для относительных путей к изображениям. |
| `--verbose` | Подробный вывод во время выполнения. |

## Применение метода к данным

После построения метода используется скрипт `evaluate.py`. Он читает сохранённый JSON-конфиг, вычисляет только те базовые компоненты, которые были выбраны на этапе построения, и сохраняет итоговые оценки качества.

Пример запуска:

```bash
uv run python evaluate.py \
  --config quality_method_config.json \
  --path-to-data-description-for-evaluation test.csv \
  --output quality_scores.json \
  --gpu 3 \
  --device cuda \
  --verbose
```

### Основные параметры `evaluate.py`

| Параметр | Описание |
|---|---|
| `--config` | Путь к JSON-конфигу, полученному после запуска `build_method.py`. |
| `--path-to-data-description-for-evaluation` | Путь к CSV-файлу, для которого нужно посчитать качество отчётов. |
| `--output` | Путь для сохранения результата. По умолчанию `quality_scores.json`. |
| `--gpu` | Номер GPU. |
| `--device` | Устройство вычислений, например `cuda` или `cpu`. |
| `--image-root` | Корневая директория для относительных путей к изображениям. |
| `--verbose` | Подробный вывод во время выполнения. |
